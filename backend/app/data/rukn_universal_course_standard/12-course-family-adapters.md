# 12 COURSE FAMILY ADAPTERS

## Adapter rule

Select one primary family and any necessary secondary families
Store the selection in `PEDAGOGY_ADAPTER`

The universal standard controls truth depth progression language projects and quality
The adapter controls what practice evidence and independence mean for this skill

The adapter must also define how invisible episodic pressure grows in this family

```yaml
family_progression:
  real_tension_sources: []
  payoff_evidence:
  next_dependency_logic:
  escalation_axis: []
  earned_peak_condition:
```

Use capability pressure native to the domain
Do not borrow theatrical tension from entertainment formats
The universal rule still requires a complete payoff inside each lesson and forbids answer withholding

Do not select an adapter from the course title alone
Select it from the observable end performance

## Production and creative adapter

Use when the learner must create an artefact such as a design video article product page photograph or edited asset

Mandatory capability loop

1. Interpret the brief and constraints
2. Select a direction for a reason
3. Build an editable first version
4. Inspect against functional evidence
5. Diagnose weakness
6. Revise without destroying strong parts
7. Adapt to another context or format
8. Package and deliver professionally

Mandatory evidence

- Source or starter state
- Decision record
- Work in progress
- First version
- Critique evidence
- Revised version
- Editable source and delivery output

Do not judge only visual or stylistic taste
Do not let tool fluency substitute for decision quality

## Operational marketing and media adapter

Use when the learner runs a process whose result appears through later signals

Mandatory capability loop

1. Define the business or operational objective
2. Separate known facts assumptions and unknowns
3. Choose a hypothesis and measurement signal
4. Configure or execute the action
5. Check implementation quality
6. Read early and mature signals without overclaiming
7. Diagnose the likely cause
8. Decide what to keep change stop or investigate
9. Communicate the decision and uncertainty

Mandatory evidence

- Brief and constraints
- Decision rationale
- Setup or campaign structure
- Measurement plan
- Simulated or real signal set
- Diagnosis
- Next decision

Never invent performance numbers
Never teach that one metric proves causality by itself
Use simulated datasets when real spend is unavailable and label them internally
Teach platform interfaces as current examples of a transferable decision process

## Sales service and live communication adapter

Use when performance depends on interaction with another person

Mandatory capability loop

1. Read the situation and goal
2. Listen or gather information
3. Choose a response function
4. Perform naturally
5. Notice the other party signal
6. Adapt rather than recite
7. Close or hand off appropriately

Mandatory practice

- Model performance
- Controlled response
- Branching scenarios
- Objection or misunderstanding
- Timed or live attempt
- Targeted feedback
- Second attempt

Do not turn the course into memorised scripts only
Reusable phrases support judgement but do not replace it

## Language adapter

Use when the target performance is comprehension speech writing pronunciation or interaction in another language

Define functions rather than topic vocabulary alone

Examples of functions include requesting clarification explaining a problem managing a lesson negotiating scope or responding to feedback

Mandatory learning loop

1. Understand the meaning and context
2. Notice form pronunciation and register
3. Retrieve without seeing the answer
4. Use in controlled variation
5. Use in a realistic interaction
6. Receive specific correction
7. Repeat with changed context
8. Recycle after spacing

Mandatory evidence

- Comprehension check
- Spoken or written production
- Pronunciation or register target where relevant
- Error diagnosis
- Corrected second performance
- Transfer to a new scenario

Do not let Arabic explanation consume most practice time
Do not teach isolated lists without retrieval and use
Do not call recognition speaking ability
Keep target language phrases exact and natural

## Programming technical and no code adapter

Use when the learner builds operates debugs or maintains a technical system

Mandatory capability loop

1. Explain the system model and environment
2. Build the smallest working path
3. Observe actual behaviour
4. Introduce realistic complexity
5. Trigger or inspect a failure
6. Diagnose with evidence
7. Fix and test
8. Refactor or organise for maintainability
9. Secure configure and deploy within scope
10. Document handoff and recovery

Mandatory evidence

- Reproducible environment
- Working artefact
- Failing case
- Diagnostic evidence
- Passing tests or verification
- Versioned source
- Deployment or inspectable package when promised

Do not read code aloud line by line
Explain state flow assumptions tradeoffs and failure evidence
Verify current library and platform behaviour from primary documentation
Do not expose credentials or teach insecure shortcuts as production practice

## Analytical data and decision adapter

Use when the learner evaluates evidence estimates prices forecasts prioritises or chooses between uncertain options

Mandatory capability loop

1. Define the decision
2. Audit the available data
3. Separate fact assumption and estimate
4. Choose method and units
5. Calculate or analyse with an audit trail
6. Stress test assumptions
7. Interpret uncertainty
8. Communicate recommendation and boundary

Mandatory evidence

- Source data
- Unit and scope checks
- Assumption ledger
- Reproducible method
- Sensitivity or alternative case
- Decision with confidence boundary

Never manufacture a missing price quantity statistic or source value
Do not present model output as source evidence
Use escalation or human approval where the decision has material commercial safety legal or financial consequence

## Knowledge authority and religious adapter

Use when the course teaches a body of authoritative knowledge interpretation or practice rather than a paid production workflow

Mandatory capability loop

1. Establish source authority and terms
2. Understand the claim in context
3. Distinguish text interpretation opinion and application
4. Correct a real misunderstanding
5. Retrieve and explain accurately
6. Apply within the stated boundary
7. Know when not to infer and when to refer to qualified authority

Mandatory evidence

- Accurate source use
- Contextual explanation
- Misconception diagnosis
- Retrieval or explanation task
- Bounded application case

Do not force portfolio language business framing or artificial projects onto this family
Use practice such as explanation classification recitation analysis teaching simulation or case judgement according to the promise
For religious content preserve the requested etiquette and terminology without mixing ceremonial formal language into otherwise natural speech in a way that sounds artificial
Never fabricate a quotation attribution consensus or personal spiritual experience

## Physical performance adapter

Use when the learner performs movement craft equipment handling or another embodied action

Mandatory capability loop

1. Establish safety and start position
2. Observe the full movement or process
3. Isolate critical components
4. Perform slowly with feedback
5. Detect unsafe or ineffective form
6. Correct one variable
7. Perform under realistic speed or load
8. Know stop and escalation conditions

Mandatory evidence

- Clear demonstration angles
- Safety boundaries
- Observable form criteria
- First attempt
- Specific correction
- Second attempt

Spoken explanation cannot replace visual demonstration and supervised practice where safety requires it

## Mixed course adapter

When a course combines families assign each capability one primary adapter

Example internal mapping

```yaml
- capability_id:
  primary_adapter:
  secondary_adapter:
  required_practice_form:
  required_evidence:
```

Do not average adapters into generic teaching
A marketing course may use analytical decisions operational execution communication with a client and production of a report
Each part must use its proper evidence

## Adapter gate

Block architecture when

- The selected adapter does not match the end performance
- Practice evidence comes from another family and cannot prove the skill
- A mixed course uses one project type for every capability
- Explanation is counted as performance
- Tool completion is counted as professional judgement
- Recognition is counted as live language ability
- A simulated result is presented as real
- A knowledge course is forced into irrelevant career project conventions
