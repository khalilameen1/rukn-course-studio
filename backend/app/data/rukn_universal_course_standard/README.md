# RUKN UNIVERSAL SKILL COURSE AUTHORING STANDARD

Machine targeted instructions for generating a complete Rukn course map then a production ready lecturer teleprompter from the first full run

Standard version `1.7-integrated-readiness-and-attachment-self-evidence`

This package is an open domain course authoring control system
It may generate a course in any field when the promised performance can be defined researched taught and represented honestly
Its final product is one teleprompter document containing the lecturer speech and compact projects between non final modules
It is not an LMS specification or a production operations manual

## Instruction precedence

Resolve conflicts in this order

1. Safety verified domain truth and current authoritative requirements
2. Explicit user product decisions audience decisions and output constraints
3. This standard
4. Verified course specific source material
5. Reference creator style calibration
6. Unverified user or source claims treated as hypotheses
7. Model defaults

Style never overrides facts
Current facts may inform research without becoming volatile recorded content
Default counts never override the capability needed to fulfil the course promise

## Final product boundary

The sole user visible course artefact is one DOCX containing only

- Course title
- Optional subtitle
- Module headings
- Reel headings
- Lecturer spoken text
- One concise spoken project after every non final module
- One unlabelled earned module bridge close at each non final boundary

The final document must not contain platform operations research notes production directions internal schemas or volatile current prices plans and rates

Projects exist only between modules
The final module ends with a complete resolution and never receives another project or forward bridge

## Definition of success

The output succeeds only when all of the following are true

- The stated learner can enter without hidden prerequisites
- A learner who completes the teaching and projects can perform the promised real task at the stated level
- The promised performance is decomposed into authentic episodes and capability units before module design
- Each capability receives a defensible primary adapter only the supporting adapters it genuinely needs and all active context overlays
- Mixed courses preserve the native modelling practice correction transfer and evidence requirements of every active capability
- Course titles and broad family labels never substitute for capability level composition
- An unlisted capability receives a practitioner grounded custom adapter rather than a forced weak match generic explanation or rejected domain
- Every reel changes an action decision diagnosis perception interaction explanation or mental model
- The course contains domain depth unavailable from generic social media summaries
- Research combines authoritative truth with practitioner attempts friction failures corrections tradeoffs and completion or handoff reality
- The course covers every required stage the learner will meet in the promised real performance episode from the first input through completion and likely recovery
- No missing middle forces the learner to invent an unintroduced action perception decision phrase access step interaction correction escalation or handoff
- Course duration and reel count are derived from required competence rather than brevity optics or copied counts
- Every standalone teaching reel is between one and five spoken minutes
- The language sounds like a credible practitioner speaking naturally to the target learner
- Terms match capable people in the target market
- Tool teaching is subordinate to durable principles and remains useful when tools change
- Rapidly expiring prices subscriptions market rates rankings limits plan details and similar content are absent from the recorded script
- Each non final module has one compact project requiring approximately ten to twenty five minutes and one to three uploads
- Every standard boundary project requires the learner to apply the most important completed module capability through an attachment whose decisive result is visible while the final boundary project integrates the promise critical capability set
- Every attachment passes an internal self evidence check showing that its decisive result can be inspected from the submitted file itself without hidden context external links unseen process
- The last project before the final module is a compact integrated readiness proof covering most core capabilities and every promise critical capability whose absence would invalidate the readiness claim
- Project text tells the learner only what to do and upload and never exposes rubrics evaluator prompts scores feedback logic or post upload behaviour
- Every non final module ends with an earned close that completes the current capability and exposes a real next pressure without naming the next module
- The final module introduces no new promise critical standalone capability that would require another formal project
- The final module diagnoses refines transfers packages communicates or closes already practised integrated performance then resolves the course promise without an artificial teaser
- Lessons and module closes create momentum through causal capability progression rather than theatrical performance answer withholding or fake suspense
- The exported document contains only allowed headings lecturer speech and inter module projects
- Teleprompter layout is visually readable sentence by sentence
- No fatal or serious quality gate remains open
- The delivered file is the result of completed drafting review rewrite evergreen scan and rendering passes rather than an exposed first draft

Passing structural validators without satisfying these conditions is failure

## Version 1.7 change

This version closes the final proof gap without adding a project after the final module
The last genuine boundary project becomes a compact integrated readiness proof rather than another isolated checkpoint
It reuses prior learner work or a bounded supplied case so it can represent most core capabilities and every promise critical capability without becoming a long capstone

The final module is now a resolution module
It may diagnose compare refine transfer package communicate deliver or hand off the integrated work using capabilities already introduced and practised
It may not introduce a new promise critical standalone capability whose honest proof would require another upload
When such a capability is discovered the architecture must move it before the last boundary project or narrow the promise

This version also adds an internal attachment self evidence check
The check asks only whether the submitted image screenshot or file directly contains the decisive visible audible readable structural or recorded runtime result needed to understand the learner action
It does not create a rubric score pass fail rule evaluator prompt feedback contract safety workflow or any post upload instruction
Nothing from this internal check appears in the lecturer DOCX

## Version 1.6 change

This version makes universality capability based rather than title or course family based
The system decomposes the promised work into authentic performance episodes then assigns every capability a primary adapter only the supporting adapters it genuinely requires and any active context overlays
Mixed composition is the normal case

The reusable adapter library now covers knowledge perception and quality inspection production operational case handling measured marketing live sales and service language use teaching and facilitation technical systems analytical decisions and physical performance
Authority safety regulation privacy tools stakeholders multilingual delivery accessibility and volatility operate as overlays rather than forcing unrelated module structures

The library is explicitly open ended
When a capability does not fit a reusable pattern the system must derive a bounded practitioner grounded custom adapter from the observable action inputs decisions signals failures correction loop transfer conditions independence standard and honest attachment evidence
It must not reduce an unfamiliar field to explanation only

This allows courses such as sales language for call centre work customer support teacher preparation technical support design and other mixed domains to retain the correct learning cycle for every part
A work language course must teach language comprehension and production inside the actual role workflow interaction diagnosis tool documentation escalation and handoff required by its promise
A teaching course must include learner response diagnosis correction retry and transfer rather than subject explanation alone

The real workflow concept is generalised to the authentic performance episode
A course may trace a client workflow call service case teaching session decision process production task physical action authoritative application or another real use sequence without forcing irrelevant business framing

The sole visible output remains one camera ready lecturer DOCX with compact projects only between non final modules

## Version 1.5 change

This version narrows the generated product to its intended use
The system builds the course map then exports only the lecturer teleprompter and compact projects between non final modules

It removes all project evaluation and LMS implementation design from course authoring and final delivery
A project still produces a directly understandable image screenshot or file but the script only tells the learner what to make and upload

It adds a mandatory module boundary close after every inter module project
The close completes the current module exposes a true professional limitation or consequence and creates natural desire to continue without saying in the next module or using artificial drama

It also makes evergreen durability a hard export rule
Exact current service prices subscriptions market rates plan details volatile limits rankings temporary trends and similar short lived information are replaced by stable decision methods or clearly hypothetical examples

## Version 1.4 retained change
